Perform an accessibility (a11y) audit of the UI.

Identify:

Missing or incorrect ARIA attributes (labels, roles, live regions)

Interactive elements that are not keyboard-focusable or have broken focus order

Images and icons without alternative text or aria-hidden where decorative

Forms: missing labels, error announcements, required-field indication

Color contrast and reliance on color alone for meaning

Focus traps in modals/dialogs and missing escape behavior

Fix using semantic HTML and ARIA where needed. Prefer native elements over custom widgets.

❗ Preserve UI behavior and visuals
❗ Do not redesign—improve accessibility of existing UI
❗ If the repo has an a11y linter or config, follow it

Goal: keyboard-usable, screen-reader-friendly, and no obvious contrast/label gaps.
